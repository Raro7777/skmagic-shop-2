## 01. 전체 대시보드
**경로**: 본사 슈퍼관리자 로그인 → 사이드바 `📊 전체 대시보드`

![](00-dashboard.png)

### 주요 영역
**① 상단 KPI 4종**
- 이번 달 정산 GMV (정산 N건, 협력점 송금 합계)
- 활성 협력점 수 (DB 활성 상태 기준)
- 이번 달 신규 lead (평균 일평균)
- 본사 수수료 = GMV − 협력점 송금 (설치비사은품 환원금 합산)
- 회신 대기 (lead) — 인증실패·수정요청·미응답 합계

**② 전체 GMV 추이 (최근 14일)** — GMV(정산) · 본사 수수료 수익 · 신규 lead 세 라인 차트

**③ 지역별 운영** — 활성 지역별 협력점 수·당월 매출

**④ 협력점 순위** — 신규 lead·정산 건수·송금액 기준 정렬

**⑤ 빠른 진입** — 인증 처리 / 설치 완료 처리 / 이번 달 정산 / 운영 이상감지

## 03. 인증 처리
**경로**: 사이드바 `🔍 인증 처리`

![](02-verify.png)

협력점이 `apply_submitted` 한 신청서를 본사가 검증 → `verify_pending` 큐에서 처리.

**액션**:
- ✅ **인증 완료** → `install_pending` 으로 전이
- ❌ **인증 실패** → `verify_failed` 회송 (사유 메모 필수, 텔레그램 알림 자동)
- ✏ **수정 요청** → `verify_revise` 회송 (협력점이 신청서 수정 후 재제출)

## 04. 설치 완료 처리
**경로**: 사이드바 `📦 설치 완료 처리`

![](03-installs.png)

`install_pending` lead 를 본사 SK매직과 일정 조율 → 설치 확인 후 `install_done` 처리.

- ✅ **설치 완료** → `install_done` → 자동 chain `settle_pending` (정산 큐 진입)
- ❌ **설치 취소** → `install_cancel` (사유 필수, Settlement 자동 cancelled)

## 25. 환수 관리
**경로**: 사이드바 `🔄 환수 관리`

![](24-refunds.png)

정산 송금 후 환수 (refund) 진행:
- `refund_pending` → `refund_progress` → `refund_done`
- 한도: partnerCommission (영업점수수료 전액)
- 환수 사유 + 금액 입력 → 시작
- refund_done 은 되돌릴 수 없음 (감사 보존)
